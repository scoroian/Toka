Contrato TodayViewModel — selector

Lista de hogares del usuario
homes: List<HomeDropdownItem>

Cada item:
id, name, emoji/icon
role: MemberRole — Propietario / Admin / Miembro
pendingCount: int — tareas pendientes hoy en ese hogar
isSelected: bool — hogar actualmente activo
Estado del dropdown
El dropdown es estado puramente visual — no pertenece al ViewModel.
La pantalla lo gestiona localmente con un bool _dropdownOpen.
El ViewModel solo expone la lista y la acción de cambio.
Acción de cambio de hogar
selectHome(String homeId)
Cambia currentHomeProvider internamente.
El resto del ViewModel (tareas, contadores) se recalcula automáticamente.
Botones crear / unirse
No necesitan lógica en TodayViewModel.
Son navegación pura: el screen hace context.go(AppRoutes.createHome)
o context.go(AppRoutes.joinHome) directamente.
Nuevo campo necesario en HomeDropdownItem:

pendingCount: int — requiere que el ViewModel lea los contadores del dashboard de cada hogar, no solo del activo.

Opción A: Leer homes/{id}/views/dashboard de cada hogar en tiempo real → costoso.
Opción B: Solo marcar si hay ≥1 pendiente (punto rojo), leyendo un campo denormalizado hasPendingToday: bool en el documento de membresía.

¿Prefieres el número exacto (Opción A) o solo el punto rojo indicador (Opción B)?
